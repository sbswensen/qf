QF Page

This page contains all the links to the features of QF.
The links here are only accessible by the administrator.

Users And Roles

  1. Roles
    For the creation, listing and deletion of roles.
  2. Users And Roles Association
    A listing of the users and the roles attached to the users. It further provides links to edit the association.

Document Structures and Aliases

  1. New Document Structure
    A document structure is a collection of data that leads to creation of form pages. This creates a new document structure.
  2. List Document Structures
    This list the document structures and provides links to create, view document structure, view created lists and delete document structures.
    In the view page of document structures you have links to edit the document structure, edit its permissions and add approvals framework to it.
  3. Style One of document Aliases
    This is the first style of creating document aliases. This feature given many new aliases to a single document structure. There is complete control over the naming of the aliases.
  4. Style Two of Document Structure Aliases
    This is the second style of creating document structure aliases. This uses a prefix or suffix to create multiple aliases on one or many document structures.

Buttons

  1. Create Button
    This creates a new button which is shared across document structures and their aliases. The button would point to a link which would run the actions of the button. The administrator would write the receiving link.
  2. List Buttons
    This list the buttons created and some details about the buttons. It also contains links to delete buttons.

Roster

  1. New Roster
    This creates a roster. Roster is used for routine tasks.
    The roster would need a document structure which would define the participants in the roster to be created. For comfort sake, there are no roster permissions. Instead it uses the create and view of the underlying document structure.
  2. List Rosters
    This list contains the rosters that have been created in an installation and is only accessible to the admin. It points to the links to fill roster, view all roster fillings and deletion of rosters.
